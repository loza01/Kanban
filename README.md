# Setup of Product Catalogue on Kanban Board (Kabana)

This repository was specifically created to show the setup of a Kanban Board for a **Product Catalogue system** as part of a Product Owner/Manager (PO) Role Examination. The goal is to showcase backlog management, user story mapping, and workflow visualisation using GitHub Projects.

## Setup Process

I applied the following steps to replicate the board used in this project:

### 1. Enable GitHub Issues
I ensure the **Issues** tab is enabled in my repo settings to serve as the foundation for your backlog items.

### 2. Create User Stories
I draft the functional requirements as individual Issues. Each story should focus on delivering specific value to the user within the catalogue system.

### 3. Initialise the Project Board
Open **Projects** tab in GitHub and create a new project using the **Board** layout.

### 4. Configure Workflow Columns
I seet up the following columns to visualise the flow of value:
*   **Backlog:** All new stories start here.
*   **Ready for Dev:** Stories that are fully defined and prioritised by the PO.
*   **In Progress:** Tasks currently being worked on.
*   **In Review / QA:** Items undergoing testing against acceptance criteria.
*   **Done,** Completed and verified features.
*   I have added Lable, estimation, assigned to myself as well

### 5. Link & Prioritize
Connect the Issues to the Project Board and stack-rank them in the **Backlog** and **Ready for Dev** columns based on business value and urgency.
