# Product Catalog Kanban Board (Kabana)

This repository was specifically created to show the setup of a Kanban Board for a **Product Catalog system** as part of a Product Owner (PO) Role Examination. The goal is to showcase backlog management, user story mapping, and workflow visualization using GitHub Projects.

## Setup Process

Follow these steps to replicate the board used in this project:

### 1. Enable GitHub Issues
Ensure the **Issues** tab is enabled in your repository settings to serve as the foundation for your backlog items.

### 2. Create User Stories
Draft the functional requirements as individual Issues. Each story should focus on delivering specific value to the user within the catalog system.

### 3. Initialize the Project Board
Go to the **Projects** tab in GitHub and create a new project using the **Board** layout.

### 4. Configure Workflow Columns
Set up the following columns to visualize the flow of value:
*   **Backlog:** All new stories start here.
*   **Ready for Dev:** Stories that are fully defined and prioritized by the PO.
*   **In Progress:** Tasks currently being worked on.
*   **In Review / QA:** Items undergoing testing against acceptance criteria.
*   **Done:** Completed and verified features.

### 5. Link & Prioritize
Connect the Issues to the Project Board and stack-rank them in the **Backlog** and **Ready for Dev** columns based on business value and urgency.
